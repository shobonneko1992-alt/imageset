# 文字打ち替えAIアプリ

画像内の文字をOCRで自動検出して編集・書き換えできる、PWA対応のWebアプリです。

## ローカルでのセットアップ

```bash
npm install
npm run dev
# → http://localhost:3000
```

本番ビルドを試す場合:
```bash
npm run build
npm start
```

## Vercelへのデプロイ

### 方法A: GitHub経由（推奨）
1. このフォルダの内容をGitHubリポジトリにpush
2. https://vercel.com にGitHubアカウントでログイン
3. 「Add New Project」→ 対象リポジトリを選択 →「Deploy」
4. 数分で `https://プロジェクト名.vercel.app` が発行される

### 方法B: Vercel CLIで直接デプロイ
```bash
npm install -g vercel
cd image-editor
vercel
```
ログインを求められたらブラウザで認証し、あとはプロンプトに従うだけです。

設定ファイル（`next.config.js`等）はそのまま使えるよう調整済みなので、追加設定は不要です。

## 機能一覧

### OCR解析
Tesseract.js（CDN経由で動的読み込み）による文字認識。日本語・英語・中国語(簡体)・韓国語に対応。検出した各行をテキストレイヤーとして抽出し、周辺の背景色を自動で吸い取って「座布団」として敷くことで元の文字を隠します。

### テキスト編集
レイヤーをタップして選択すると、移動ハンドル（紫・上部）・削除ハンドル（赤・右上）・リサイズハンドル（緑・右下）が出現します。ツールバーではフォント・サイズ・太字/斜体・文字色・背景色・透明度に加え、横書き/縦書きの切替、行間・字間調整、文字の縁取り（色・太さ）、ドロップシャドウ（色・ぼかし・オフセット）を設定できます。

### Undo / Redo
ドラッグ・リサイズ・テキスト編集・スタイル変更などの操作を履歴として記録し、ヘッダーの←→ボタンまたはCtrl/Cmd+Z（戻す）、Ctrl/Cmd+Shift+ZまたはCtrl+Y（やり直す）で操作を戻せます。

### キーボードナッジ
レイヤー選択中に矢印キーで1px単位、Shift+矢印キーで10px単位の微調整ができます（テキスト編集中は無効）。

### 整列・スナップ
レイヤーをドラッグすると、他のレイヤーの端・中央や画像全体の端・中央に近づいた際に赤いガイド線が表示され、自動的に吸着します。

### 画像トリミング
「トリミング」ボタンでクロップモードに入り、8方向ハンドルで矩形選択して切り出せます。確定すると画像サイズが更新され、既存のテキストレイヤーも新しい原点に合わせて自動的に座標が補正されます。

### 画像補正（明るさ・コントラスト）
「補正」ボタンのスライダーで画像全体の明るさ・コントラストを調整できます。OCR実行時は補正後の画像を解析するため、文字をくっきりさせることで認識精度を上げられます。保存時も補正が反映された状態で書き出されます。

### 高解像度エクスポート
「保存」ボタンで、画面表示のCSSサイズに関係なく**元画像の解像度のまま**PNGを生成します。テキストの座標・サイズ・装飾・画像補正がすべてピクセルパーフェクトで合成されます。

### PWA対応
`manifest.json` と Service Worker（`public/sw.js`）により、スマートフォンのホーム画面に追加してスタンドアロン表示できます。

## ファイル構成

```
app/
├── layout.tsx       # PWAメタデータ・ビューポート設定・Service Worker登録
├── page.jsx         # アプリ本体（状態管理・OCR・Canvas描画・全UIを1ファイルに集約）
└── globals.css      # グローバルスタイル
public/
├── manifest.json     # PWAマニフェスト
├── sw.js             # Service Worker（最小構成）
├── icon-192.png      # アプリアイコン
└── icon-512.png      # アプリアイコン
```

`page.jsx` は意図的に単一ファイルにまとめています（OCRサービス、Canvas描画ヘルパー、Undo/Redo用フック、各UIコンポーネントをすべて内包）。TypeScriptの厳格な型チェックの対象から外すため拡張子は `.jsx` にしていますが、Next.jsのApp Router上で問題なく動作します。

## 注意事項

- Tesseract.jsは初回OCR実行時にCDN（cdnjs）からモデルをダウンロードするため、数秒かかることがあります
- アイコン画像は簡易的なプレースホルダーです。本番公開する際は好みのデザインに差し替えてください
- iOS Safariでは`<input type="file">`をボタンのrefから`click()`で発火する方式を採用しているため、通常のタップで問題なく動作します
